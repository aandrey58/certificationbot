# Excel Creator

Десктопная утилита (PyQt5): по JSON с вопросами создаёт **новый лист** в существующем `.xlsx` и заполняет его в формате, который ожидает парсер основного бота (`parser.py` в корне репозитория).

Имя листа в Excel совпадает с **названием области аттестации** в Telegram-боте (как в подсказке в окне программы).

## Зачем это нужно

- Единожды подготовить или дополнить банк вопросов в Excel без ручного копирования столбцов.
- Сохранить совместимость с форматом, который читает `parser.py`: заголовки как в шаблоне (порядковый номер, вопрос, до шести вариантов с колонками проверки, нормативная основа).

## Требования

- Python 3.10+ (рекомендуется 3.11)
- Windows (ниже — PowerShell; на Linux/macOS пути к venv другие)

## Установка и запуск

Из **корня репозитория** `certificationbot`:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r excel_creator\requirements.txt
python excel_creator\main_window.py
```

Либо из папки `excel_creator` (если зависимости уже в активном окружении):

```powershell
cd excel_creator
python main_window.py
```

В окне:

1. Укажите **целевой** Excel-файл (`.xlsx`) — файл будет **изменён** (добавится лист).
2. Укажите JSON-файл с массивом вопросов.
3. Введите **название листа** (например, `Б 2.2.`). Если лист с таким именем уже есть, программа создаст `Название (1)`, `Название (2)` и т.д.
4. Нажмите **Создать**.

## Формат JSON

Ожидается **массив объектов**. Поля (как в примерах `А1.json`, `Б 2.4..json` в этой папке):

| Поле | Тип | Описание |
|------|-----|----------|
| `number` | число или строка | Номер пункта (первый столбец листа; заголовок такой же, как в шаблоне для бота) |
| `question` | строка | Текст вопроса |
| `options` | массив строк | До **6** вариантов ответа (лишние обрезаются) |
| `correct` | массив строк **или** одна строка | Правильные ответы; сопоставление идёт **по точному тексту** варианта из `options` |
| `reference` | строка | Попадает в колонку «Нормативная основа вопроса» |

Пример одного элемента:

```json
{
  "number": 1,
  "question": "Текст вопроса?",
  "options": ["Вариант А", "Вариант Б"],
  "correct": ["Вариант А"],
  "reference": "ст. 1 НПА"
}
```

Для каждого варианта в столбец проверки записывается `+`, если текст варианта входит в `correct`, иначе `-`.

## Как получить JSON из Word (обязательно через ChatGPT)

Инструкция, чтобы корректно извлечь **жирный** правильный ответ из `.docx` и получить JSON для утилиты:

1. **Соберите банк вопросов и ответов в Word-файл** (`.docx`). (обязательно)
2. **Выделите правильный(е) ответ(ы) жирным шрифтом**. (обязательно)
3. **Загрузите этот Word-файл именно в ChatGPT**.  
   Важно: *не Qwen и не DeepSeek* — из этих трёх **только ChatGPT** корректно считывает жирное выделение в `.docx`. (обязательно)
4. **Вместе с файлом отправьте текст промпта** из `excel_creator/data/Промт для ChatGPT.txt`. (обязательно)
5. **Сохраните ответ ChatGPT как `.json`** (UTF‑8) и используйте этот файл в `.exe` утилиты (или при запуске `python main_window.py`).

## Связь с ботом

После создания листа:

1. Сохранённый `.xlsx` положите в каталог **`data/`** в корне проекта бота (см. основной `README.md` в репозитории) **или** загрузите файл через интерфейс Telegram-бота (кнопка **«📤 Загрузить Excel»**, доступна только администраторам).
2. Перезапустите бота (или выполните загрузку через Telegram) — парсер подхватит листы и создаст/заполнит таблицы в PostgreSQL.

## Сборка в `.exe` (Windows)

Зависимость `pyinstaller` указана в `requirements.txt`.

Из корня репозитория, с уже установленными пакетами из `excel_creator\requirements.txt`:

```powershell
pyinstaller --onefile --noconsole -n excel_creator excel_creator\main_window.py
```

Исполняемый файл: `dist\excel_creator.exe`.

**Иконка окна** задаётся через Qt-ресурс `icon_rc.py` (`:/icon.ico`). Если в собранном `.exe` иконка не отображается, может понадобиться явно подключить ресурс в параметрах PyInstaller или оформить `.spec` — см. документацию PyInstaller для PyQt5.

## Файлы в этой папке

| Файл | Назначение |
|------|------------|
| `main_window.py` | Точка входа, UI и логика заполнения листа |
| `icon_rc.py` | Скомпилированный ресурс Qt (иконка) |
| `requirements.txt` | PyQt5, openpyxl, pyinstaller |
| `А1.json`, `Б 2.4..json` | Примеры входного JSON |

## Ограничения и замечания

- Поддерживается только **`.xlsx`** (диалог выбора файла фильтрует именно его).
- JSON должен быть **корневым массивом** `[]`, не объектом `{ ... }`.
- Элементы массива, которые не являются объектами, пропускаются.
